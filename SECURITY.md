# Security Policy

## Supported versions

Security fixes are provided for the latest minor release. The `main` branch is not a
stability promise.

## Reporting a vulnerability

Use GitHub private vulnerability reporting for `rthgit/zmlc-router`. Do not open a
public issue containing secrets, exploit details, or private prompts. Include the
affected version, platform, reproduction, and expected impact.

ZMLC never requires an API key for its Codex plugin path. If credentials appear in a
trace or release artifact, treat that as a security defect.
